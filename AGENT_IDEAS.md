# Agent 创意记录

## 趣味化 Agent 方向

### Agent 性格皮肤
- 不是单纯换提示词语气，而是“运行事件人格”。
- 同一个 YYZ_CLAW 可以切换成冷面审计官、暴躁修 Bug 师、阴阳怪气评审、赛博管家、福尔摩斯模式等。
- 不同人格改变行为偏好：有的更爱先验证，有的更爱质疑需求，有的主动写事故报告，有的会吐槽烂代码。

### Agent 情绪条
- 情绪不是假装人格，而是根据运行状态生成压力值。
- 连续 tool error、上下文过长、用户频繁打断、测试失败都会让压力上涨。
- 压力变化可以触发冷静模式、崩溃边缘、杀疯了模式、请给我十秒钟模式等状态。

### 任务 BOSS 战
- 把复杂任务渲染成打 BOSS。
- BOSS 血条代表剩余问题数，护盾代表未确认风险，技能代表当前 blocker。
- 每完成一个验证、修掉一个 bug，BOSS 掉血；build/test 全过就是击杀。

### Agent 内心独白层
- 不是暴露完整 chain-of-thought，而是可控的表层心理活动。
- 例如：“我怀疑这里不是前端问题，而是落库没发生。”
- 例如：“这个 diff 有点危险，我先不碰配置。”
- 目标是让用户感到 agent 在场，同时不泄露复杂推理链。

### 任务称号系统
- 每次完成任务自动生成一个离谱称号。
- 例如：队列驯兽师、SQLite 外科医生、SSE 拆弹专家、前端排版赎罪券。
- 可以给会话、agent、用户一起打标签，增加记忆点。

### Agent 互相留纸条
- 主 agent、子 agent、hook、调度器之间不只发消息，也可以留下便利贴。
- 例如：“我怀疑这个 API 有坑，后来者注意。”
- 例如：“不要再查这个文件了，我查过。”
- 前端可以做成一面案板，不作为普通聊天消息渲染。

### 会话鬼魂
- 历史失败任务会变成鬼魂提醒。
- 当用户或 agent 准备做类似危险操作时弹出提醒。
- 例如：“上次你在这里把 models 提交进去了。”
- 例如：“上次这个队列没落库，刷新后丢了。”
- 它比普通记忆更有项目幽灵感。

### Agent 口头禅进化
- 根据项目历史自动生成 workspace 黑话。
- 例如：“这不是排队，这是伪装成插入的排队。”
- 例如：“先别动 config。”
- 例如：“蓝色消息又开始作妖了。”
- 久了之后 agent 会像真的和项目一起生活过。

### 审判庭模式
- 对一个方案开庭。
- 检方 agent 专门指出漏洞，辩方 agent 维护方案，法官 agent 最后裁决。
- 不是多智能体乱聊，而是固定流程。
- 最终输出判决书：可做、暂缓、死刑。

### 任务遗物馆
- 每个大任务结束后自动保存遗物。
- 遗物可以包括关键 diff、最坑的 bug、最终命令、用户一句名言、agent 的一句吐槽。
- 以后打开项目可以看到一排战利品，让系统更有陪伴感。

## 当前最值得优先尝试
- 会话鬼魂
- 任务 BOSS 战
- Agent 互相留纸条
