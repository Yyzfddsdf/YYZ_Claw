yyz先生，换一批更“真有用”的：

1. **会话分支**
   从任意一条消息 fork 出新会话，保留前文，后面重新跑。这个对调试 prompt、试不同方案、回到模型跑偏前非常有用，比快照更像产品功能。

2. **消息级重跑**
   点某个 assistant 回答旁边的“重跑”，删除它之后的 assistant/tool 结果，从同一条 user 重新生成。这个是 AI IDE 必备能力。

3. **局部上下文锁定**
   用户可以把某几条消息锁定为“永远带上”，不会被压缩删掉。适合需求约束、关键结论、接口约定。
   
4. **会话文件夹/任务看板**
   会话太多以后，用文件夹或看板组织：待做、进行中、已完成、归档。自动化、remote、普通 chat 都能挂进去。

5. **产物索引**
   扫描工作区里由 agent 创建/修改的文件，生成一个“本会话产物”列表。点进去看 diff、打开文件、复制路径。这个比模型总结可靠，因为基于真实文件变化。

6. **跨会话搜索**
   搜索所有历史消息、tool result、文件名、错误信息。你现在 history 已经落库了，这个功能会很实用。

7. **会话模板**
   不是 skills。是用户创建“启动一类任务的首条消息 + 推荐身份 + 推荐自动化 + 可选工作区范围”。比如“代码审查模板”“前端 UI 修改模板”“远程值守模板”。

8. **运行中接管**
   模型跑到一半时，用户可以点某个 tool call 或等待状态旁边“接管/跳过/补结果”。比如工具卡死、浏览器操作失败时，不用整轮重来。

我这次最推荐：**会话分支 + 消息级重跑 + 跨会话搜索**。这三个不是装饰，是真会每天用的基础能力。
