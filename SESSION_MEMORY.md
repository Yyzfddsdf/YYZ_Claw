# SESSION MEMORY

## 上一步实际完成了什么
- 已删除主工具 `subagent_inspect`，`subagent_list` 只保留现存子智能体的基础信息，避免把子智能体历史再塞进主上下文
- 后台 run 的自动压缩现在会发 `compression_started / compression_completed`，前端按会话跟踪压缩态，并允许压缩中继续输入排队、切换会话、创建/Fork 其他会话
- 停止按钮、本地运行态回收、清空历史接口、调度器可见消息瘦身都已落地
- 左侧历史列表已改成主会话分组视图：有父会话的子智能体对话不再平铺，而是收进主会话下面，通过按钮展开/收起查看
- 主会话顶部那排平铺的子智能体 chip 已移除，避免同一批子会话在 UI 里出现两套入口
- 已修复子智能体分组按钮“点了没反应”的问题：自动展开只在切入新的子会话时触发，不再覆盖用户手动折叠
